2026-04-04

python中的文件操作

打开文件：open（）

打开文件之后，会根据你给的mode（模式）的不同来进行不同的操作，open的参数如下：
open(文件路径,mode="你要干啥",encoding="文件的编码格式")

文件路径

文件路径主要分为两种：
1.相对路径
在python中，相对路径是以当前正在运行的py文件所在的文件夹为基准的
常用写法：

  • a.txt表示当前文件夹下的a.txt
  • ./a.txt和上面一样，.表示的就是当前目录
  • ../b.txt表示上一级文件夹中的b.txt
  • ../data/c.txt表示上一级的data文件夹里的c.txt

只需要记住../指的是退出这个文件夹，比如下图中正在运行的是py字符串.py,那么python序列.py这个文件就可以直接表示为python序列.py或者./python序列.py
如果要访问aaa这个文件夹里的a.py文件,就要先退出当前文件夹，然后进入a.py文件，路径就是../aaa/a.py

2.绝对路径
绝对路径指的是从磁盘根目录开始查找的路径，比如C盘，D盘，E盘，F盘等，比如C:/sylar/xxx.txt

mode

mode是字符串参数，由操作类型（r/w/a/x）+数据类型（t/b）组成的，默认值是rt（只读文本）
1.核心操作类型(必选一个)

模式 英文全称 作用 关键特性
r read 只读 文件必须存在，不然会报错
w write 只写 文件不存在就创建，存在就清空原来的所有内容
a append 追加 文件不存在就创建，存在就写入内容到文件末尾
x exclusive 独占创建 文件不存在就创建，存在直接报错

2.数据类型（可选，默认t）

模式 英文全称 作用 适用场景
t text 文本模式 读写字符串(文字、代码、配置文件等)
b binary 二进制模式 读写非文本文件(图片、视频、音频、压缩包等)

encoding

encoding仅在**文本模式(t)**下生效，二进制模式(b)不需要这个参数
作用：告诉python用什么编码规则把字符和二进制互相转化，用于解决中文乱码
1.常用编码格式

编码 特点 使用场景
utf-8 通用编码，支持全球语言 绝大多数文本文件
gbk/gb2312 中文专用编码 Windows系统的记事本、旧中文文档
ascii 仅支持英文 纯英文文件

2.关键注意点：

  • 读写编码必须一致
  • windows默认编码：gbk,MAC/Linu默认编码：utf-8
  • 不指定encoding时，python会用系统默认编码，建议手动指定utf-8

文件操作写法

推荐用with语句，它会在代码结束后自动关闭文件，防止资源泄露

关于strip():

  • line读取文件时，末尾会包含一个换行符\n(比如”Hello\n”)
  • 使用.strip()后，会得到"Hello"()(去掉了换行符)
  • 如果行首或者行尾有空格，也会一并去掉

.write()和.writelines()的区别:.writelines()可以批量写入多个字符串

路径处理

可以用pathlib(Python3.4+内置)，代码更清晰，跨平台兼容更好
